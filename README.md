## Naming Conventions:

To keep the project consistent and easy to maintain, the following naming conventions will be used throughout the repository:

- File and folder names: use lowercase letters only
- Words in file/folder names: separated with underscores (_)
- Python files, variables, and functions: use snake_case
- Class names: use PascalCase
- Constants: use uppercase letters with underscores (UPPER_CASE)
- Database table and column names: use lowercase with underscores (snake_case)
- No spaces or special characters in file, folder, or object names
- Meaningful and descriptive names should always be preferred over vague abbreviations

These conventions are intended to improve readability, consistency, and collaboration across the project.
